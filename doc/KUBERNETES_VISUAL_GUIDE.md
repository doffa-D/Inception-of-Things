## What is Kubernetes?

> "Kubernetes is a system that **manages containers** - it runs them, monitors them, restarts them if they crash, and scales them up or down based on demand."

### Simple Definition:

```
Docker = Run ONE container
Kubernetes = Manage MANY containers across MANY machines
```

### Key Features:

| Feature | What it does |
|---------|--------------|
| **Self-healing** | Restarts crashed containers automatically |
| **Scaling** | Add/remove containers based on load |
| **Load balancing** | Distributes traffic across containers |
| **Rolling updates** | Update app without downtime |
| **Declarative** | You describe desired state, K8s makes it happen |

---

## K3s vs K3d:

| Feature | K3s (Parts 1 & 2) | K3d (Part 3) |
|---------|-------------------|--------------|
| **Runs on** | Real VMs (Vagrant) | Docker containers |
| **Nodes** | Real machines | "Fake" containers |
| **Setup** | Heavy (VMs) | Light (containers) |
| **Use case** | Production-like | Development/testing |
| **Speed** | Slower to start | Very fast |

## Quick Reference Commands:

```bash
# Cluster
k3d cluster list                    # List clusters
k3d cluster create <name>           # Create cluster
k3d cluster delete <name>           # Delete cluster

# Namespaces
kubectl get ns                      # List namespaces
kubectl create namespace <name>     # Create namespace

# Pods
kubectl get pods                    # List pods in default namespace
kubectl get pods -n <namespace>     # List pods in specific namespace
kubectl get pods -A                 # List all pods in all namespaces

# Deployments
kubectl get deployments             # List deployments
kubectl scale deployment <name> --replicas=3  # Scale

# Services
kubectl get services                # List services

# Ingress
kubectl get ingress                 # List ingress
kubectl describe ingress <name>     # Show details
```
